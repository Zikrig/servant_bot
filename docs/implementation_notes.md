# Implementation Notes

## Компоненты

- `src/main.py` — FastAPI приложение, webhook endpoint, startup init.
- `src/bot_service.py` — оркестрация личного меню, мастера сценариев и автоответов в чатах.
- `src/scenario_manager.py` — бизнес-правила сценариев и валидация.
- `src/storage.py` — SQLite доступ к данным.
- `src/db.py` — DDL и индексы.
- `src/panel_renderer.py` — текст и inline-клавиатура панели.
- `src/telegram_client.py` — обертка над Bot API.

## Callback протокол

- `menu:auto` / `menu:help` / `menu:main` — навигация по главному меню.
- `auto:add` — начать создание сценария.
- `wiz:*` — шаги пошагового мастера.
- `we:*` — редактирование выходных и праздничных дат.
- `wh:*` — редактирование рабочего времени.
- `sc:toggle:{id}` — включить/выключить сценарий.
- `sc:delask:{id}` — запрос подтверждения удаления.
- `sc:delete:{id}` — подтвержденное удаление.

Все callback-команды короче лимита 64 байта.

## Критерии приемки MVP

1. `/start` в личном чате показывает приветствие и главное меню.
2. Добавление сценария работает через пошаговый мастер.
3. При включении сценария автоматически отключается предыдущий активный.
4. Удаление требует подтверждения кнопкой.
5. Бот отвечает в управляемом чате только если владелец сценария сам не ответил до истечения `steel_pause`.
6. Повторные ответы ограничиваются параметрами сценария (`not_answer_twice`, `hot_pause`, выходные, рабочее время).
7. Приложение запускается через `docker compose up --build`.
