# Telegram API Notes (May 2026)

Этот файл фиксирует технические решения для текущей реализации.

## Используемые методы

- `setWebhook` — прием апдейтов в webhook-режиме.
- `sendMessage` — отправка сообщений и панели.
- `editMessageText` — перерисовка панели без лишних сообщений.
- `answerCallbackQuery` — обязательный ответ на инлайн-кнопки.

## Лимиты и ограничения

- `callback_data` в inline-кнопках: 1-64 байта.
- В Guest Mode бот видит только сообщение с `@mention` и ответы в этом треде.
- Telegram не хранит бизнес-state сценариев: состояние хранится в нашей БД.
- Старые кнопки могут оставаться в истории, поэтому коллбеки сделаны идемпотентными.

## Актуальные возможности Bot API (весна 2026)

- В Bot API 9.5+ есть `sendMessageDraft` (стриминг), но в этом MVP используется классический `sendMessage`.
- Переход на стриминг можно добавить в следующей итерации без изменения доменной модели.
